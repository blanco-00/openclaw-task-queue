# 提案：添加智能任务拆分功能

## Why

当前任务队列插件缺少任务拆分能力：
1. 用户只能手动创建单个任务
2. 复杂的任务（如"重构整个项目"）不知道如何拆
3. 缺乏归档和清理机制

需要像 OpenSpec 一样，具备智能拆分 + 原子化的能力。

## What Changes

1. **添加 task_decompose 工具**
   - 接收复杂任务描述
   - AI 分析后拆分为可执行的小任务
   - **支持依赖关系** (dependsOn)
   - **支持执行顺序** (order)
   - 不知道如何拆时返回问题让用户确认

2. **原子化任务拆分**
   - 每个子任务独立可执行
   - 包含明确的输入输出
   - 支持任务依赖声明
   - 支持执行顺序控制

3. **添加归档功能**
   - 自动归档已完成的任务
   - 定期清理过期数据
   - 支持手动触发归档

4. **添加清理功能**
   - 清理 cancelled/dead 状态任务
   - 清理超过 N 天的已完成任务
   - 可配置保留策略

## Capabilities

### New Capabilities
- `task-decompose`: 智能任务拆分
- `task-archive`: 任务归档
- `task-cleanup`: 任务清理

### Modified Capabilities
- 无

## Impact

- **API影响**: 新增 3 个工具函数
- **数据库影响**: 添加 archive_at, cleaned_at 字段
- **用户影响**: 更智能的任务管理体验
