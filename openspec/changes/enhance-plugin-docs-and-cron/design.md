# 设计文档：增强插件文档和内置定时任务

## Context

任务队列插件的核心价值是实现 AI 自驱闭环：
- 用户提交任务 → AI 自动拆分 → 定时领取 → 执行 → 反馈

当前用户不知道怎么使用这个模式，需要更好的文档和开箱即用的配置。

## Goals / Non-Goals

**Goals:**
1. 更新文档，详细描述工作流模式
2. 添加内置 cron 配置
3. 确保并发安全

**Non-Goals:**
- 不修改核心任务队列逻辑
- 不改变现有 API

## Decisions

### D1: 内置 Cron 配置方式
**决策:** 在插件配置中添加 cron 相关选项，自动注册服务
**理由:** 用户安装插件后，配置一下就能用

### D2: 并发控制策略
**决策:** 使用 CAS 原子操作防止重复领取
**理由:** SQLite 支持原子更新，已有此机制

## Open Questions

1. **Q: 如何平衡「定时领取」和「用户主动领取」?**
   - 方案1: cron 只处理高优先级任务
   - 方案2: cron 只在用户指定时才激活
   - 建议默认方案1

2. **Q: 如何限制并发?**
   - 使用配置中的 concurrency 字段
   - cron 任务检查当前运行数
