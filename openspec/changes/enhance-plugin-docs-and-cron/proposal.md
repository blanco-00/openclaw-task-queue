# 提案：增强插件文档和内置定时任务

## Why

当前插件存在以下问题：
1. 用户不知道如何使用这个插件完成「提交任务 → 拆分 → 定时领取 → 执行」的完整工作流
2. 需要用户手动配置 cron 任务，学习成本高
3. 没有描述这个插件的核心价值：AI 自驱闭环

## What Changes

1. **更新插件文档**
   - 详细描述「任务拆分 → 定时领取 → 执行」的工作流模式
   - 说明如何使用这个模式构建长运行 agent
   - 添加常见使用场景示例
   - 添加配置项说明

2. **添加内置定时任务配置**
   - 插件内置默认 cron 配置
   - 用户安装插件后自动可用
   - 支持配置覆盖

3. **优化并发控制**
   - 确保 cron 任务不重复领取任务
   - 添加任务状态追踪机制
   - 添加并发限制检查

## Capabilities

### New Capabilities
- `plugin-cron-config`: 内置定时任务配置
- `workflow-documentation`: 工作流文档增强

### Modified Capabilities
- 无

## Impact

- **代码影响**: 添加 cron 配置模块
- **文档影响**: 更新 plugin-task-queue.md
- **用户影响**: 降低使用门槛，安装即用
